## 结构体内存对齐规则

    1.结构体的第一个成员用房放在结构体位置偏移量为0的位置。
    2.结构体成员从第二个成员开始总是放在偏移量为一个对齐数的整数倍处
        对齐数=编译器磨人的对齐数和变量自身大小的较小值
        linux - 没有默认对齐数
        vs - 默认对齐数是8
    3.结构体的总大小必须是各个成员的对齐数中最大那个对齐数的整数倍
    4. 如果嵌套了结构体的情况，嵌套的结构体对齐到自己的
    最大对齐数的整数倍处，结构体的整体大小就是所有最大
    对齐数（含嵌套结构体的对齐数）的整数倍。

节省空间

    创建结构体时 把小字节变量放在一起
    struct S1
    {
    	char c1;
    	int i;
    	char c2;
    };//对齐后 空间占用12字节
    struct S2
    {
    	char c1;
    	char c2;
    	int i;
    };//对齐后 空间占用8字节
修改默认对齐数

    #pragma pack(8)  修改
    #pragma pack()   恢复 
    建议设置2的次方 
offsetof 计算结构体变量的偏移量

    printf("%u\n",offsetof(struct S1,c1);//0
    printf("%u\n",offsetof(struct S1,ci);//4
    printf("%u\n",offsetof(struct S1,c2);//8

**结构体传参建议传址**  提升性能
