## 源码、反码、补码
> **内存中存的是补码**


    - 源码
        直接进二进制按照正负数翻译成二进制
    - 反码
        源码符号位不变，按位取反
    - 补码
        反码+1
    
    补码 -> 源码 ：补码 -1取反
    补码 -> 源码 ：补码 取反+1 


## 大小端存储模式


    小端字节存储： vs2019
        把一个数字的 低 位字节放在内存的 低 地址处
                     高 地址存放在内存的 高 地址处
    大端字节存储：
        把一个数字的 低 位字节放在内存的 高 地址处
                     高 地址存放在内存的 低 地址处

写一个程序 判断大端小端
```
int main()
{
    int a = 1;
    char* p = &a;
    if(*p == 1)
    {
        printf("小端\n");
    }
    else
    {
        printf("大端\n");
    }
    return 0;
}
```

- **IEEE 754 规定**
> - **对于符号位M**
> IEEE 754规定，在计算机内部保存M时，默认这个数的第一位总是1，因此可以被舍去，只保存后面的xxxxxx部分。比如保存1.01的时

> - **对于指数E** 
> **首先E是无符号整数(unsigned int)**  
> 因为E 算出来可能是负(列入0.5的科学计数法是1.0*2^-1)  
> 所规定E+127  或者 E+1023 再转换成二进制存进E 的位置  

> - **E不全为0或不全为1**
> 还原时再加上-127 或者 -1023 得到真实值，  
> 再将有效数字M前加上第一位的1。
> - **E为全0**  
> 这时，浮点数的指数E等于1-127（或者1-1023）即为真实值 
> 有效数字M不再加上第一位的1，而是还原为0.xxxxxx的小数  
> 这样做是为了表示±0，以及接近于0的很小的数字。
> - **E全为1**
> 这时，如果有效数字M全为0  
> 表示±无穷大（正负取决于符号位s）


    (-1)^S * M * 2^E
    (-1)^s表示符号位，当s=0，V为正数；当s=1，V为负数。
    M表示有效数字，大于等于1，小于2。
    2^E表示指数位。
    ---
    32为浮点数float
            0 00000000 00000000000000000000000
      (1bit)S   E(8bit)           M(23bit)
      最高的1位是符号位S
      接着的8位是指数E
      剩下的23位为有效数字M
    
    64为浮点数double
            0 00000000 00000000000000000000000
      (1bit)S   E(11bit)           M(52bit)
      最高的1位是符号位S
      接着的8位是指数E  
      剩下的23位为有效数字M
